# Requisitos do Projeto Integrador Transdisciplinar II

## 1. Introdução

Este documento apresenta os requisitos funcionais e não funcionais da aplicação desenvolvida por Mohamed Radwan como parte do Projeto Integrador Transdisciplinar em Engenharia de Software II.

## 2. Objetivo

Desenvolver uma aplicação web para cadastro e listagem de usuários, utilizando arquitetura MVC, com front-end em HTML/CSS/JavaScript e back-end em Java com Spring Boot.

## 3. Requisitos Funcionais

- RF01: O sistema deve permitir o cadastro de usuários com nome e e-mail.
- RF02: O sistema deve listar todos os usuários cadastrados.
- RF03: O sistema deve validar os campos obrigatórios no front-end.
- RF04: O sistema deve persistir os dados no banco de dados MySQL.
- RF05: O sistema deve exibir mensagens de sucesso ou erro após ações do usuário.

## 4. Requisitos Não Funcionais

- RNF01: O sistema deve seguir o padrão de arquitetura MVC.
- RNF02: O sistema deve ser responsivo e acessível.
- RNF03: O sistema deve tratar erros de forma clara e segura.
- RNF04: O sistema deve ser versionado com Git e hospedado no GitHub.
- RNF05: O sistema deve ser testado com JUnit no back-end.

## 5. Tecnologias Utilizadas

- HTML, CSS, JavaScript
- Java com Spring Boot
- MySQL
- JUnit
- Git e GitHub

## 6. Escopo Futuro

- Autenticação de usuários
- Dashboard administrativo
- Exportação de dados em PDF
